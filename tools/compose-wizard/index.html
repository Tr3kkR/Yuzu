Welcome to the Yuzu Compose Wizard

This wizard will walk you through creating a customised docker-compose.yml and .env file for your Yuzu deployment.

Why use this? The default compose file works, but it has hardcoded values and several known gotchas. This wizard helps you avoid common mistakes and generates a production-ready configuration.

What you'll configure:

  • Yuzu version, admin credentials, and ports
  • PostgreSQL storage substrate (bundled or external), data dir, and ClickHouse analytics
  • Prometheus & Grafana observability stack
  • Yuzu Gateway (Erlang/OTP) settings

What you'll get:

  • docker-compose.yml — fully commented, ready to run
  • .env — environment variables for easy reconfiguration
  • Quick-start commands to get going immediately

Default Certificates
The server generates a per-install CA and the https/agent/gateway leaf certs on first boot (ensure_default_certs). Nothing to supply. The cert directory is persisted (see below) so the CA survives a docker compose down && up — otherwise every recreate mints a fresh CA and breaks already-enrolled agents' trust.

Operator Certificates
Mount your PEM material into the container. The paths below are container paths; the wizard bind-mounts a host ./certs directory read-only at /etc/yuzu/certs. Put your files there, or edit the mount in the generated compose.
The server refuses to start with operator certs and no CA (mTLS client verification is mandatory). To deliberately disable client-cert verification you must run the server with --insecure-skip-client-verify + YUZU_ALLOW_INSECURE_TLS=1 yourself.
Plaintext (insecure)
⚠️ Development only. Emits --no-tls --no-https, overriding the secure-by-default behaviour. The agent↔server channel is unencrypted — never internet-expose port 50051 in this mode (command fan-out = fleet-RCE risk).

Data & Storage

PostgreSQL is the server's storage substrate (ADR-0006/0008). As of v0.12 the server stores its state in PostgreSQL, not SQLite. The server needs a database to talk to — choose a bundled local container or point at an external/managed instance below.
Bundled uses the release-pinned yuzu-postgres image (PostgreSQL 16 + pgvector). External points the server at a managed database.
Bundled PostgreSQL

Yuzu Gateway

The Yuzu Gateway (Erlang/OTP) provides agent connection pooling, circuit breaking, and hash-ring routing. It sits between agents and the Yuzu server.
TLS + gateway isn't generated yet. The secure gateway↔server topology (mutual-TLS upstream, shared certs) depends on server features still in flight (PKI go-live, #1314) — the wizard would otherwise emit a stack that can't boot. With the gateway enabled you must pick Plaintext on the Core step (and keep :50051 off the public internet). For a secure gateway today, follow deploy/docker/docker-compose.reference.yml + gateway/config/sys.config.prod.
