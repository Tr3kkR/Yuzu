B — platform-aware SKIP (9)

Two halves; do both.

  1. YAML audit: the 9 Windows-only defs currently declare platforms: too broadly. Fix each YAML to declare platforms: [windows].
  2. Runner classification: before dispatch in instructions_runner.py exercise():
host = {"Darwin": "darwin", "Linux": "linux", "Windows": "windows"}[platform.system()]

C — parity work (the headline)

  • crossplatform.process.fetch — add macOS branch (proc_listpids, proc_pidpath, sysctl). Linux branch already exists.
  • device.wifi.list_networks — replace airport (removed in macOS 14+). Try system_profiler SPAirPortDataType first (no privilege); fall back to wdutil info with sudoers entry. Linux: nmcli -t -f SSID,BSSID,SIGNAL device wifi list.
  • If you discover tar.export, agent_logging.get_log, or event_logs.errors have stale macOS branches once buckets A/F clear, file an issue per plugin.

D — type: query (2)

Two-line fix. In content/definitions/discovery.yaml and content/definitions/deployment.yaml, change type: query → type: question for device.discovery.scan_subnet and server.deployment.list_jobs. Rebuild; verify with:

Orthogonally, consider patching server/core/src/instruction_store.cpp to accept query as an alias for question — fixes the class, not just these two instances.

E — tar.export UNION ALL (1)

Real plugin bug. In agents/plugins/tar/src/tar_plugin.cpp (or sibling), the export action composes a UNION ALL across TAR tables; one branch's column count diverges. Align columns; SQLite emits the error pre-exec so success is verified just by dispatch returning rc=0.

F — empty-result rc=1 (2)

  • device.agent_logging.get_log: when get_log_file_path() returns empty, return status|empty rc=0 instead of status|error|agent log file not found. "No rows" is not "error".
  • device.event_logs.errors: legitimate error lines start with error|; runner currently treats that as failure. Fix runner-side — trust the agent's rc rather than parsing column-0 of the body.

    --output /tmp/instructions-outcomes.json

# 3. Inspect the fail list
python3 -c "
import json; d=json.load(open('/tmp/instructions-outcomes.json'))
print('outcomes:', {o['outcome'] for o in d['outcomes']})
for o in d['outcomes']:
    if o['outcome'] in ('fail','error'):
        print(' ', o['definition_id'], '|', o.get('note',''))
"
Probe a single failing instruction against the live agent 
curl -s -c /tmp/cookies.txt -X POST http://localhost:8080/login \
