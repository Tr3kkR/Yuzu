Network quality — fleet

Continuous per-connection quality measured on each endpoint from kernel counters (no packet capture, no flow export), rolled up across the fleet. The headline isn’t a chart of latency — it’s where the pain is: because network quality lives in the same on-device warehouse as CPU/disk and process/service events, we can show whether a slow app co-occurs with a device or app problem. A network-only tool can’t — it never sees the device.

This is counting, not blaming. “Also under device pressure” means the two facts co-occur on the same box at the same time — it is not a verdict that the device caused the slowness (memory pressure can look like a network fault). The causal verdict (weighted most-likely cause) is a deliberate overlay: a contained deterministic rule for automation, and an agentic colleague for deep-dive. v1 ships the evidence; the overlay names the cause.

Collection & privacy
Per-connection destinations reveal who/what people talk to, so this is opt-in, usage-class telemetry (netqual_enabled, off by default, separate kill-switch). At the edge: corporate-infra destinations are kept; user-browsing destinations are bucketed or dropped before anything is stored; per-app destination cardinality is top-N capped so a browser opening thousands of sites can’t bloat the warehouse. Raw rows never leave the device; per-device/app views are an on-demand, permission-gated, audited query (network.device.view).
v1 = the evidence surface. This Overview ships: fleet-now cards (thin heartbeat facts), the co-occurrence panel (counted, no verdict), and the worst-devices drill with co-occurring flags inline. Not on this page (deliberate): any “it’s the network” blame label, and any fleet-wide per-app breakdown (per-app stays in the device warehouse, reached on the Devices drill — keeps the heartbeat thin). Ladder (named, not built): network as a scope-DSL dimension (net.rtt_p90 > 200) → scope-walking (“what do the slow-RTT devices share?”) → the verdict overlay (deterministic rule + agentic) → Guardian closed-loop.
