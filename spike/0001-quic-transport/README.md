# Spike — msquic ↔ quicer interop (#376 PR 0)

Throwaway code. The migration plan is in `docs/adrs/0001-quic-transport-msquic-quicer.md`;
this directory exists only to validate the four pass criteria from the
"Spike validation gate (PR 0)" section before any production code is
written. Evidence captured under `docs/spike-results/0001-quic-transport/`.

## Layout

```
cpp-server/      C++ msquic server (CMake + isolated vcpkg manifest)
erlang-client/   Erlang quicer client (rebar3)
certs/           Self-signed cert pair generated by scripts/gen-certs.sh
scripts/         gen-certs.sh, run-all.sh — drive the four scenarios
```

## How to run

```
bash scripts/gen-certs.sh           # one-shot
bash scripts/run-all.sh             # build, run all four scenarios, capture evidence
```

`run-all.sh` writes per-scenario logs to `docs/spike-results/0001-quic-transport/`.

## Pass criteria (from ADR-0001)

1. TLS 1.3 handshake succeeds with self-signed cert pair
2. Bidi stream survives 30 s of two-way 1 KiB writes with zero loss
3. Half-close from either end is detected by the other within 1 s
4. Flow control: writer that blocks for 5 s on a slow reader resumes cleanly when reader catches up

If any fail: halt migration, update ADR-0001 to Rejected, file new ADR.
