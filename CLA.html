How to sign

You can sign this CLA in either of these ways:

  1. Automated (preferred, once the CLA bot is live) — open a pull request against this repository. The CLA workflow at .github/workflows/cla.yml will comment on your PR with a link to a signing page. Your GitHub account is recorded as a signatory. This workflow is disabled by default and requires the CLA_REPO_ACCESS_TOKEN secret to be provisioned by ops; until it is active, use the manual path below.

  2. Manual — open a pull request that appends a line to the "Signatories" section of this file in the format:

- <Full Name>, <GitHub handle>, <date YYYY-MM-DD>

Corporate signatories add a second line noting the employer and, where applicable, the contact who holds authority to bind the employer:

- Corporate: <Company>, authorised representative <Name> <email>, <date>
